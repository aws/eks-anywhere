  The permissions needed by the EKS Anywhere vSphere user

The permissions needed by the EKS Anywhere vSphere user are just short of full administrative access. For example:
 
## Virtual machine

### Configuration

* Change configuration
* Add existing disk
* Add new disk
* Add or remove device
 
### Advanced configuration

* Change CPU count
* Change memory
* Change settings
* Configure raw devices
* Extend virtual disk
* Modify device settings
* Remove disk
* Create from existing
* Remove
 
### Interaction

* Power off
* Power on
 
### Provisioning

* Deploy template
 
## Datastore

* Allocate space
* List datastore
* Low level file operations
 
## Network

* Assign network
 
## Resource Pool

* Assign a virtual machine to resource pool
